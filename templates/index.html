Undirected vs Directed Graphs

The graphs we saw till now were undirected , that is the edges are bidirectional, with no direction associated with them. Hence, the graph can be traversed in either direction. There is another type called directed in which The edges of the graph represent a specific direction from one vertex to another. When there is an edge representation as (V1, V2), the direction is from V1 to V2. The first element V1 is the initial node or the start vertex. The second element V2 is the terminal node or the end vertex.

Graph Representations

Adjacency matrix:
The adjacency matrix of a graph is the matrix A is labelled as 1 or 0 according to whether the vertex vi is adjacent to the vertex vj or not , that is it is labelled 1 if there is an edge between vertex i and vertex j. In case of weighted graph we use the weights instead of marking presence of edge by 1.




Adjacency List:

Adjacency list is a linked representation. In this representation, for each vertex in the graph, we maintain the list of its neighbours. It means, every vertex of the graph contains list of its adjacent vertices. We have an array of vertices which is indexed by the vertex number and for each vertex v, the corresponding array element points to a singly linked list of neighbours of v.

Real life applications:

Have you ever thought about how Google Maps or your GPS is able to show you the shortest path to your destination? Applications such as Google Maps are able to do that using a class of algorithms called Shortest Path Finding Algorithms by assigning each location as a node and paths as edges with distances as weight. Similarly Facebook and other social networking sites use graphs to suggest friends. Each person is a node and they have an edge to their friends and neighbours of these friend vertices can be suggested as friends to user. These are just few of the many applications this data structure has to offer.
